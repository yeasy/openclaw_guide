# 架构深入（选读）

本章是第一章"核心概念速览"的进阶延伸，适合完成基础入门后希望深入理解 OpenClaw 运行机制的读者。

如果你刚刚安装完 OpenClaw 并想快速上手，可以先跳过本章，等实际遇到排障场景时再回来查阅。

## 本章内容

- **[架构全景与核心对象](arch_and_concepts.md)**
  拆解四层逻辑架构与五大核心对象（Gateway、Node、Agent、Session、Tool）的完整设计，以及基于 π 的极简运行底座。

- **[请求流转与分层排障](request_flow.md)**
  追踪一条消息从渠道到模型再到用户的完整生命周期，掌握"由外到内"的分层排障策略。

## 学习目标

完成本章阅读后，你将能够：
1. 理解 Gateway 作为单一控制平面如何串联外部渠道与内部执行层
2. 将五大核心对象（Gateway、Node、Agent、Session、Tool）映射到具体的配置和排障场景
3. 在系统出现问题时，快速定位到正确的层级进行排查
