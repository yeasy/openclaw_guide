## 13.2 实战案例：客户支持智能体

### 13.2.1 问题定义与场景约束

某B2B SaaS公司日均接收500多条客户支持请求，分布在电子邮件和在线聊天等多个渠道。现有6人客服团队平均响应时间为4小时。公司希望部署一个支持智能体来自动处理常见问题并加速响应。

核心需求：

- **FAQ自动回复**：自动解答密码重置、账单查询、API错误等常见问题
- **情感感知升级**：检测负面情绪，自动升级到人工客服处理
- **工单系统集成**：无法自动解决的问题自动创建Zendesk工单
- **多语言支持**：同时支持中文和英文

系统约束：

- 必须与现有Zendesk系统集成
- PII（个人身份信息）必须在发送给LLM之前掩盖
- 负面情感检测后升级必须在2分钟内完成

### 13.2.2 最小闭环：单工具单渠道跑通

我们从最简单的配置开始：一个仅能通过Webhook查询知识库的支持智能体。以下是完整的`openclaw.json`配置文件。

```json
{
  "gateway": {
    "port": 18789,
    "auth": {
      "requirePairing": true
    }
  },
  "agents": {
    "customer-support": {
      "name": "客户支持智能体",
      "description": "处理客户咨询，自动解答常见问题",
      "model": "claude-opus-4-6",
      "tools": ["knowledge_base_search"],
      "systemPrompt": "你是一个专业的客户支持代理。你的职责是：\n1. 理解客户的问题\n2. 在知识库中检索相关解决方案\n3. 提供清晰、准确的答案\n\n关键原则：\n- 如果不确定答案，请诚实地说明\n- 始终保持礼貌和同情心",
      "memory": {
        "strategy": "session_with_summary",
        "maxTokens": 4000,
        "compactAfterTurns": 10
      }
    }
  },
  "tools": {
    "knowledge_base_search": {
      "enabled": true,
      "type": "vector_db",
      "provider": "qdrant",
      "endpoint": "http://qdrant:6333",
      "indexName": "customer_kb",
      "searchTopK": 3,
      "minScore": 0.6
    }
  },
  "channels": {
    "webhook": {
      "type": "webhook",
      "port": 18789,
      "path": "/support/query",
      "agentId": "customer-support"
    }
  }
}
```

**验收：** 通过Webhook发送“如何重置密码”，验证机器人返回了知识库中的步骤。

### 13.2.3 关键配置拆解

#### 升级触发规则

添加以下配置到`agents[customer-support]`中，定义何时将对话升级到人工处理。详见[第 7.3 章](../07_multi_agent/7.3_routing_basics.md)。

```json
"escalation": {
  "triggers": [
    {
      "type": "sentiment_negative",
      "threshold": -0.7,
      "action": "escalate_to_human"
    },
    {
      "type": "request_human",
      "keywords": ["人工", "客服", "经理"],
      "action": "escalate_to_human"
    },
    {
      "type": "confidence_low",
      "threshold": 0.4,
      "action": "ask_for_confirmation"
    }
  ]
}
```

当用户的负面情绪评分低于-0.7或明确请求人工客服时，系统会自动升级。

#### Zendesk工单集成

在`tools`部分添加以下配置，使智能体能够创建工单。详见[第五章](../05_tools_skills/README.md)和[第 12.2 章](../12_extension_engineering/12.2_custom_tools.md)。

```json
"zendesk_ticket_manager": {
  "enabled": true,
  "type": "zendesk",
  "subdomain": "${ZENDESK_SUBDOMAIN}",
  "email": "${ZENDESK_EMAIL}",
  "apiToken": "${ZENDESK_API_TOKEN}",
  "timeout": 10000,
  "fieldMapping": {
    "subject": "title",
    "description": "description",
    "priority": "priority",
    "tags": ["support", "automated"]
  }
}
```

在`agents[customer-support].tools`中添加`zendesk_ticket_manager`。

#### PII掩盖处理

> **注意**：以下配置为功能示意。实际 PII 处理在 OpenClaw 中通过结构化日志事件和自定义工具层实现。详见[第 8.1 章](../08_automation_ops/8.1_hooks.md)官方 Hook 系统。

在工具调用前进行 PII 掩盖，可在知识库搜索工具的包装层实现。参考实现：

```json
"knowledge_base_search": {
  "type": "builtin:knowledge_base",
  "description": "在内部知识库中搜索文档和代码注释",
  "config": {
    "index_path": "./kb/docs",
    "similarity_threshold": 0.6,
    "preprocess_patterns": [
      {
        "regex": "\\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Z|a-z]{2,}\\b",
        "replacement": "[EMAIL_MASKED]"
      },
      {
        "regex": "\\b\\d{11}\\b",
        "replacement": "[PHONE_MASKED]"
      }
    ]
  }
}
```

此配置在工具执行前掩盖电子邮件和电话号码。

#### 多语言支持

在`agents[customer-support]`中添加语言检测和本地化配置。详见[第七章](../07_multi_agent/README.md)。

```json
"language": {
  "detection": true,
  "defaultLanguage": "auto",
  "supportedLanguages": ["zh", "en"],
  "responseLocale": "auto"
}
```

系统会自动检测用户消息的语言，并用相同语言回复。

### 13.2.4 验收清单与常见失败点

验收步骤：

1. **FAQ自动回答是否工作** — 发送一个已知的FAQ问题（如“如何重置密码”），验证机器人返回了知识库中的答案。

2. **升级是否在负面情感时触发** — 发送一条愤怒的消息（如“这太糟糕了，我要找客服！”），验证系统创建了工单并升级给人工。

3. **PII是否被掩盖** — 发送包含电子邮件地址的消息，检查日志中是否显示为`[EMAIL_MASKED]`。

4. **Zendesk工单字段是否正确** — 检查Zendesk仪表板，验证自动创建的工单包含所有必需字段（标题、描述、优先级）。

常见失败及解决：

| 失败症状 | 根本原因 | 修复方法 |
|---------|--------|--------|
| 知识库检索无结果 | 嵌入向量API密钥未配置或超时 | 检查`tools[knowledge_base_search]`中的`endpoint`和认证；见第6.3.3节 |
| 升级不触发 | 阈值设置过严格或情感分析缺乏上下文 | 降低`escalation.triggers[sentiment_negative].threshold`至-0.6；确保至少有3句完整句子 |
| PII泄露到日志 | 掩盖钩子优先级低于日志钩子 | 确保`before_tool_call`钩子的`priority`为100+，运行在日志钩子之前 |
| 工单字段缺失 | `fieldMapping`配置与Zendesk自定义字段不匹配 | 在Zendesk管理界面验证字段名称，更新`zendesk_ticket_manager.fieldMapping`中的映射 |

### 13.2.5 扩展到生产时再加什么

**添加电子邮件和Zendesk聊天渠道**：将Webhook扩展为电子邮件和Zendesk原生聊天。详见[第七章](../07_multi_agent/README.md)的渠道集成部分。只需在`channels`中添加`email`和`zendesk_chat`配置即可。

**知识库向量数据库**：从内存中的简单列表升级为Qdrant或Weaviate等持久化向量数据库，以支持更大的知识库和更快的检索。详见[第 6.3 章](../06_context_memory/6.3_memory_mechanism.md)中的记忆与知识管理机制。

**性能调优与成本控制**：添加缓存策略、批量处理、请求去重和模型路由（如根据复杂度选择不同的LLM）。详见[第十四章](../14_performance_cost/README.md)的性能优化与成本控制策略。

**审计和合规日志**：实施完整的消息日志记录、加密存储和访问审计，以满足GDPR或其他数据保护法规。详见[第十一章](../11_reliability_security/README.md)的安全与合规部分。
