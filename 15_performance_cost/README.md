## 快速开始

### 第一步：诊断当前成本

```bash
# 生成成本报告
openclaw metrics export --period last_30_days --format cost_analysis

# 识别成本驱动因素
openclaw analyze --metric token_usage
openclaw analyze --metric api_calls
```

### 第二步：选择优化策略

根据成本分布选择优先级：
- 如果API 成本占 > 80%：优先实施Token 优化
- 如果基础设施成本占 > 30%：优先实施资源优化
- 如果人工成本占 > 20%：优先实施自动化

### 第三步：实施与验证

```bash
# 前置：记录基线指标
openclaw metrics baseline

# 实施优化（如启用缓存）
openclaw config set tools.*.cache.enabled true

# 监控变化（2-4周）
openclaw metrics compare --baseline 2024-03-01

# 量化收益
openclaw report --metric cost_reduction
```

## 成本监控指标

关键指标：

- **Daily Burn Rate**（日消耗）：监控是否超预算
- **Cost per Request**：衡量单位效率
- **Token Efficiency**：平均消耗Token数

- **Cost Trend**：成本变化趋势

## 常见问题

**Q: 优化会影响服务质量吗？**

A: 不会。优化主要在于更有效利用现有资源，而非削弱功能。实际上：
- Token 优化不改变功能，只是表述更精确
- 缓存加速提升用户体验
- 模型选择可按复杂度调整

**Q: 优化需要多久才能看到效果？**

A:
- 简单优化（如启用缓存）：即时见效
- 中等优化（如模型路由）：1-2周稳定
- 复杂优化（如知识库微调）：4-8周显著

**Q: 如何在成本和质量之间平衡？**

A: 参考第15.2章的场景建议。核心原则：
- 简单问题→便宜模型（Haiku）
- 中等问题→平衡模型（Sonnet）
- 复杂问题→强大模型（Opus）
