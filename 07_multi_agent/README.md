# 第七章 多渠道分发与多智能体协作

当入口从单渠道扩展到多渠道、多群与多对端后，系统最常见的问题不是模型能力不足，而是边界不清：谁负责处理、哪些入口允许触发、哪些能力属于高风险、出现问题如何回放定位。

本章的主线是把多渠道与多智能体做成可管理系统：

- 渠道策略负责收敛触发面。
- 路由绑定负责收敛所有权。
- 工具策略与沙箱约束负责收敛可执行边界。
