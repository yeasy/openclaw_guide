# 第一章：认识 OpenClaw

本章先建立一张稳定、可复用的 OpenClaw 认知地图：它解决什么问题、系统边界在哪里、核心对象如何协作、学习顺序如何安排。读完后，可以把 OpenClaw 视为一个“以 Gateway 为控制中枢、以 Agent Runtime 为执行内核、以 Channels/Tools/Memory 为能力表面”的工程系统，而不是零散命令集合。

**本章学习目标**

- 建立 OpenClaw 的产品定位与系统定位。
- 理解 Gateway、Agent、Node、Tool、Session 等核心概念及其关系。
- 明确从“能用”到“精通”的分阶段学习路径。
- 为后续安装、配置、调优与实现原理章节建立统一术语基线。

**阅读建议**

建议按 `1.1 → 1.2 → 1.3 → 1.4` 顺序阅读。若此前已使用过 OpenClaw，可重点阅读 `1.2` 与 `1.3`，并在 `1.4` 对照自身经验建立能力缺口清单。

从更宏观的视角看，OpenClaw 的框架体系可以解耦为四条核心架构主线：

- **连接机制** ：解决在复杂外部网络约束下，如何确保异构平台消息可信抵达内网，完成身份确权与绑定。
- **执行内核** ：模型与工具咬合的运转车间，负责处理动态提示词与上下文，清晰处理工具返回结果，维持深层思考。
- **治理策略** ：用基于资源与身份的角色体系与静态规则约束引擎动作，严厉限制跨界的执行授权。
- **扩展生态** ：通过标准化插件接口定义，将成熟任务模块化，实现在组件系统之间互通及分享交流。
