# 第一章 认识 OpenClaw

欢迎来到 OpenClaw 的世界。如果你曾经使用过 ChatGPT 或 Claude，你可能会惊叹于大语言模型的聪明才智；但当你试图让它们帮你“真正把活干完”（比如自动查收邮件、整理数据并发送到飞书群）时，很容易发现缺了一个能把 AI 与真实软件连接起来的系统框架。

**OpenClaw 就是这样一个能够将大语言模型与真实世界连接起来，让 AI 替你自动、安全地完成复杂任务的智能体项目。**

作为全书的开篇，本章旨在建立一张清晰、系统的“认知地图”，梳理 OpenClaw 解决的核心问题、划定它的系统边界。通过本章，你将建立对 OpenClaw 的整体认知。

## 本章内容导读

本章包括以下几个小节：

- **[1.1 大模型演进与 OpenClaw 的诞生](1.1_history.md)** 
  带你了解 OpenClaw 到底能帮你做什么。探讨大模型在落地真实业务时遇到的工程痛点，以及 OpenClaw 是如何化解这些问题的。

- **[1.2 架构与核心概念](1.2_architecture.md)** 
  用图解的方式拆解 OpenClaw 的“四层架构”与“五大核心对象”（Gateway、Agent、Node、Tool、Session）。

- **[1.3 对比其它方案](1.3_comparison.md)** 
  对比 OpenClaw 与 ChatGPT、Cursor 或是传统的自动化工具（如 Zapier）的根本区别，帮你理清思路，选对正确的工具。

- **[1.4 典型的潜在应用场景](1.4_use_cases.md)**
  结合生态能力，为你盘点个人 ChatOps、家庭自动化网关以及安全实验平台等几种硬核实战思路。

- **[1.5 本章小结](summary.md)**
  快速的知识清单和几道自测题，帮你验证自己是否“完全准备好”进入下一章的实战安装环节。

## 学习目标

完成本章的阅读后，你将能够：
1. **明确系统边界**：准确说出 OpenClaw 是什么，以及它最适合（和不适合）的工作场景。
2. **统一术语基础**：理解 Gateway、Agent、Tool、Session 等核心概念的含义，为后续的配置调优扫清沟通障碍。
3. **建立全局视角**：知道一次请求从进入系统到了解工具执行结果，大概经历了哪些模块和协作步骤。

准备好的话，让我们正式 **[开始](1.1_history.md)**！
