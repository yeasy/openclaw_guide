## 16.1 版本映射与配置字段变更记录

本章提供OpenClaw各版本间配置字段的精确映射、废弃字段的替代方案以及升级迁移指南，确保跨版本兼容性与平滑升级。

### 16.1.1 版本演进时间线

```
v0.1 (2023-Q1) → v0.8 (2023-Q4) → v1.0 (2024-Q1) → v1.2 (2024-Q3) → v1.5 (2025-Q1)
  │                │                 │                │                 │
  核心功能        多渠道支持        GA版本          企业级功能        Claude集成增强
  MVP            Lark/Slack等       稳定API        K8s支持            Cost管理
```

### 16.1.2 配置字段变更对照表

#### v0.8 → v1.0 主要变更

| 旧字段 (v0.8) | 新字段 (v1.0) | 变更类型 | 迁移说明 |
|--------------|-------------|--------|--------|
| `agent.name` | `agents.{agentId}.name` | 结构重组 | 从单Agent改为多Agent，需要转换为Map格式 |
| `tools.enabled` | `tools.{toolId}.enabled` | 结构优化 | 同上，改为按工具ID的Map |
| `channels.lark.token` | `channels.lark.appId + appSecret` | 安全升级 | 分离了认证信息，提升安全性 |
| `memory.maxSize` | `memory.maxContextTokens` | 单位统一 | 改为Token计数，更精确 |
| `failover.strategy` | `failover.model.strategy` | 结构化 | 支持工具级和模型级分别配置 |
| `logging.level` | `logging.levels` (object) | 细粒度 | 支持为不同模块设置不同日志级别 |
| `security.requireAuth` | `gateway.auth.requirePairing` | 重命名 | 移到Gateway层，概念更准确 |

#### v1.0 → v1.2 主要变更

| 旧字段 (v1.0) | 新字段 (v1.2) | 变更类型 | 迁移说明 |
|--------------|-------------|--------|--------|
| `agents.*.tools` (Array) | `agents.*.tools` + `agents.*.toolGroups` | 扩展 | 新增工具组概念，便于管理关联工具 |
| `memory.strategy` | `memory.strategy` + `memory.compactionMode` | 增强 | 新增压缩模式选项：aggressive/balanced/conservative |
| `channels.*.rateLimit` | `gateway.rateLimit` + `profiles.*.rateLimit` | 层级化 | 改为Gateway全局 + Profile单独配置两层 |
| `tools.*.policy` | `tools.*.policy` + `profiles.*.toolPolicies` | 重构 | Policy现在可在Profile层覆盖工具级配置 |
| `hooks.*.script` (string) | `hooks.*.script` (path) + `hooks.*.inline` (code) | 灵活性 | 支持外部脚本文件或内联代码 |

#### v1.2 → v1.5 主要变更

| 旧字段 (v1.2) | 新字段 (v1.5) | 变更类型 | 迁移说明 |
|--------------|-------------|--------|--------|
| `agents.*.model` (string) | `agents.*.models` (array) | 多模型 | 支持多模型fallback链，提升可靠性 |
| `gateway.port` (number) | `gateway.listeners` (array) | 扩展 | 支持同时监听多个端口/协议 |
| `channels.*.webhookPath` | `channels.*.routing.webhookPath` | 结构化 | 路由规则更灵活 |
| `memory.maxContextTokens` | `memory.contextBudget` (object) | 精细化 | 支持为不同类型上下文设置不同预算 |
| `tools.*.cache.ttl` (number) | `tools.*.cache` (object) | 结构化 | 支持更复杂的缓存策略（LRU, TTL, Size等） |

### 16.1.4 自动升级脚本

```javascript
/**
 * 版本迁移工具：自动将旧配置转换为新格式
 * 支持：v0.8 → v1.0, v1.0 → v1.2, v1.2 → v1.5
 */

const MigrationRules = {
  'v0.8_to_v1.0': {
    // 规则1：agent单体 → agents映射
    transform_agent_structure(config) {
      if (config.agent && !config.agents) {
        const agentId = config.agent.id || 'default';
        config.agents = {
          [agentId]: {
            name: config.agent.name,
            model: config.agent.model,
            tools: config.agent.defaultTools || []
          }
        };
        delete config.agent;
      }
      return config;
    },

    // 规则2：channels.lark.token → channels.lark.appId/appSecret
    transform_lark_auth(config) {
      for (const [key, channel] of Object.entries(config.channels || {})) {
        if (key === 'lark' && channel.token) {
          // token通常是appId:appSecret的格式
          const [appId, appSecret] = channel.token.split(':');
          channel.appId = appId;
          channel.appSecret = appSecret;
          delete channel.token;
        }
      }
      return config;
    },

    // 规则3：memory.maxSize → memory.maxContextTokens
    transform_memory_config(config) {
      if (config.memory && config.memory.maxSize && !config.memory.maxContextTokens) {
        // 假设maxSize是字符数，转换为Token（粗略：1字符 ≈ 0.5 Token）
        config.memory.maxContextTokens = Math.ceil(config.memory.maxSize * 0.5);
        delete config.memory.maxSize;
      }
      return config;
    },

    // 规则4：security.requireAuth → gateway.auth.requirePairing
    transform_security_config(config) {
      if (config.security && config.security.requireAuth !== undefined) {
        if (!config.gateway) config.gateway = {};
        if (!config.gateway.auth) config.gateway.auth = {};
        config.gateway.auth.requirePairing = config.security.requireAuth;
        delete config.security.requireAuth;
      }
      return config;
    }
  },

  'v1.0_to_v1.2': {
    transform_tools_to_groups(config) {
      // 旧格式：tools: ['tool1', 'tool2', ...]
      // 新格式：toolGroups: { group1: ['tool1', 'tool2'] }
      // 简单迁移：将所有工具放在'default'组
      for (const [, agent] of Object.entries(config.agents || {})) {
        if (Array.isArray(agent.tools)) {
          agent.toolGroups = { 'default': agent.tools };
          // agent.tools 保留以实现向后兼容
        }
      }
      return config;
    },

    transform_rate_limit_hierarchy(config) {
      // 旧格式：channels.*.rateLimit
      // 新格式：gateway.rateLimit + profiles.*.rateLimit
      const channelLimits = {};
      for (const [key, channel] of Object.entries(config.channels || {})) {
        if (channel.rateLimit) {
          channelLimits[key] = channel.rateLimit;
          delete channel.rateLimit;
        }
      }

      if (Object.keys(channelLimits).length > 0) {
        if (!config.gateway) config.gateway = {};
        config.gateway.rateLimit = channelLimits;
      }

      return config;
    },

    transform_memory_compaction(config) {
      // 旧格式：memory.compactOnTurns
      // 新格式：memory.compactionMode
      if (config.memory && config.memory.compactOnTurns) {
        // compactOnTurns数值较小时 → 'aggressive'
        // 数值较大时 → 'conservative'
        const turns = config.memory.compactOnTurns;
        config.memory.compactionMode = turns < 10 ? 'aggressive' : 'balanced';
        delete config.memory.compactOnTurns;
      }
      return config;
    }
  },

  'v1.2_to_v1.5': {
    transform_single_model_to_multi(config) {
      // 旧格式：agents.*.model: 'claude-opus-4-6'
      // 新格式：agents.*.models: [{ model: '...', weight: 1.0 }]
      for (const [, agent] of Object.entries(config.agents || {})) {
        if (typeof agent.model === 'string') {
          agent.models = [
            { model: agent.model, weight: 1.0, priority: 'primary' }
          ];
          delete agent.model; // 可选：保留以兼容
        }
      }
      return config;
    },

    transform_gateway_listeners(config) {
      // 旧格式：gateway.port: 18789
      // 新格式：gateway.listeners: [{ port: 18789, protocol: 'http' }]
      if (config.gateway && typeof config.gateway.port === 'number') {
        config.gateway.listeners = [
          {
            port: config.gateway.port,
            protocol: 'http',
            host: config.gateway.host || '0.0.0.0'
          }
        ];
        // 保留port以向后兼容
      }
      return config;
    },

    transform_tool_timeout(config) {
      // 旧格式：tools.*.timeout: 30000
      // 新格式：tools.*.readTimeout 和 tools.*.writeTimeout
      for (const [, tool] of Object.entries(config.tools || {})) {
        if (typeof tool.timeout === 'number') {
          tool.readTimeout = tool.timeout;
          tool.writeTimeout = tool.timeout;
          delete tool.timeout;
        }
      }
      return config;
    }
  }
};

class ConfigMigrator {
  /**
   * 自动检测当前版本并进行逐级迁移
   */
  async migrate(configPath, targetVersion = 'latest') {
    const currentConfig = this.readConfig(configPath);
    const currentVersion = this.detectVersion(currentConfig);

    logger.info(`Detected version: ${currentVersion}`);
    logger.info(`Target version: ${targetVersion}`);

    let config = currentConfig;
    const versions = ['v0.8', 'v1.0', 'v1.2', 'v1.5'];
    const fromIndex = versions.indexOf(currentVersion);
    const toIndex = targetVersion === 'latest' ?
      versions.length - 1 :
      versions.indexOf(targetVersion);

    // 逐级迁移
    for (let i = fromIndex; i < toIndex; i++) {
      const from = versions[i];
      const to = versions[i + 1];
      const migrationKey = `${from}_to_${to}`;

      if (MigrationRules[migrationKey]) {
        logger.info(`Migrating from ${from} to ${to}...`);
        const rules = MigrationRules[migrationKey];

        for (const [, rule] of Object.entries(rules)) {
          config = rule(config);
        }

        logger.info(`✓ Migration ${from} → ${to} complete`);
      }
    }

    // 验证迁移结果
    const validationResult = await this.validate(config, targetVersion);
    if (!validationResult.valid) {
      throw new Error(
        `Migration validation failed: ${validationResult.errors.join(', ')}`
      );
    }

    // 备份原配置
    const backupPath = `${configPath}.backup.${currentVersion}`;
    this.writeConfig(backupPath, currentConfig);
    logger.info(`✓ Backup saved to: ${backupPath}`);

    // 写入新配置
    this.writeConfig(configPath, config);
    logger.info(`✓ Configuration updated to ${targetVersion}`);

    return config;
  }

  /**
   * 自动检测配置版本
   */
  detectVersion(config) {
    // v1.5 标志
    if (config.agents && config.gateway?.listeners) {
      return 'v1.5';
    }

    // v1.2 标志
    if (config.agents && config.gateway?.rateLimit !== undefined) {
      return 'v1.2';
    }

    // v1.0 标志
    if (config.agents && !config.agent) {
      return 'v1.0';
    }

    // v0.8 标志
    if (config.agent) {
      return 'v0.8';
    }

    return 'unknown';
  }

  /**
   * 验证配置的合法性
   */
  async validate(config, version) {
    const errors = [];

    // 版本特定的验证规则
    const schemaFile = `./schemas/${version}.json`;
    const schema = require(schemaFile);

    const ajv = new Ajv();
    const validate = ajv.compile(schema);
    const valid = validate(config);

    if (!valid) {
      errors.push(...validate.errors.map(e => e.message));
    }

    return { valid: errors.length === 0, errors };
  }

  readConfig(path) {
    const fs = require('fs');
    return JSON.parse(fs.readFileSync(path, 'utf-8'));
  }

  writeConfig(path, config) {
    const fs = require('fs');
    fs.writeFileSync(path, JSON.stringify(config, null, 2));
  }
}

// 使用示例
const migrator = new ConfigMigrator();
await migrator.migrate('~/.openclaw/openclaw.json', 'v1.5');
```

### 16.1.5 向后兼容性说明

OpenClaw遵循以下兼容性政策：

#### 支持政策

- **主版本（Major）**：推荐在6个月内升级，超期可能存在安全风险
- **次版本（Minor）**：2个版本内保持向后兼容（例如v1.2可读v1.0配置）
- **补丁版本（Patch）**：完全兼容

```
时间轴：

v1.4 发布时，v1.2配置仍可运行（兼容期）
v1.6 发布时，v1.2配置不再支持，需要升级
```

#### 兼容性表

```
┌────────┬───────┬───────┬───────┬───────┐
│ 读取版本 │ v0.8  │ v1.0  │ v1.2  │ v1.5  │
├────────┼───────┼───────┼───────┼───────┤
│ v0.8   │  ✓   │  ✓   │  ✗   │  ✗   │
│ v1.0   │  ✓   │  ✓   │  ✓   │  ✗   │
│ v1.2   │  ✗   │  ✓   │  ✓   │  ✓   │
│ v1.5   │  ✗   │  ✗   │  ✓   │  ✓   │
└────────┴───────┴───────┴───────┴───────┘

✓: 完全兼容
✗: 不兼容（需要升级迁移）
```

### 16.1.6 版本升级步骤

```bash
#!/bin/bash
# upgrade_openclaw.sh

set -e

OLD_VERSION=$(openclaw version)
TARGET_VERSION="v1.5"

echo "OpenClaw升级指南"
echo "当前版本: $OLD_VERSION"
echo "目标版本: $TARGET_VERSION"
echo ""

# 步骤1：备份
echo "[1/5] 备份当前配置..."
cp -r ~/.openclaw ~/.openclaw.backup.$(date +%Y%m%d)

# 步骤2：自动迁移配置
echo "[2/5] 自动迁移配置文件..."
openclaw config migrate \
  --input ~/.openclaw/openclaw.json \
  --output ~/.openclaw/openclaw.json.new \
  --target-version $TARGET_VERSION

# 步骤3：验证迁移
echo "[3/5] 验证配置有效性..."
openclaw config validate ~/.openclaw/openclaw.json.new || {
  echo "ERROR: 配置验证失败，请查看上方错误日志"
  exit 1
}

# 步骤4：替换配置
echo "[4/5] 替换配置文件..."
mv ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.$OLD_VERSION
mv ~/.openclaw/openclaw.json.new ~/.openclaw/openclaw.json

# 步骤5：更新二进制
echo "[5/5] 更新OpenClaw二进制..."
openclaw upgrade --version $TARGET_VERSION

echo ""
echo "✓ 升级完成!"
echo "新配置已生成，旧配置备份在: ~/.openclaw/openclaw.json.$OLD_VERSION"
echo ""
echo "后续步骤："
echo "1. 运行 'openclaw doctor' 进行诊断检查"
echo "2. 在非生产环境进行充分测试"
echo "3. 监控日志以确认服务稳定性"
```
