## 16.3 OpenAI 与本地模型集成

OpenClaw 不绑定单一 AI 供应商。除 Claude 外，OpenAI 与以 Ollama 为代表的本地模型也是生产环境中常见的接入选择。本节介绍这两类生态的接入方式和典型配置，以及与 Claude 组合使用的混合策略。

### 16.3.1 OpenAI 模型接入

OpenClaw 通过 `openai` 供应商标识接入 OpenAI 的模型家族（GPT 系列、o 系列推理模型等）。

#### 基础配置

```javascript
{
  models: {
    providers: {
      openai: {
        apiKey: "${OPENAI_API_KEY}",
      },
    },
  },
  agents: {
    defaults: {
      model: {
        primary: "openai/gpt-4o",
        fallbacks: ["openai/gpt-4o-mini"],
      },
    },
  },
}
```

#### 使用 Azure OpenAI

企业用户通常通过 Azure OpenAI Service 调用 GPT 模型，可通过自定义 `baseUrl` 接入：

```javascript
{
  models: {
    providers: {
      openai: {
        apiKey: "${AZURE_OPENAI_API_KEY}",
        baseUrl: "https://your-resource.openai.azure.com/openai/deployments/your-deployment",
        apiVersion: "2024-02-01",
      },
    },
  },
}
```

#### OpenAI 接入要点

| 能力 | 说明 |
|------|------|
| **工具调用** | Function Calling 与 OpenClaw 工具系统兼容，格式自动转换 |
| **结构化输出** | `response_format: { type: "json_schema" }` 可配合 OpenClaw 输出解析 |
| **o 系列推理模型** | `openai/o3`、`openai/o4-mini` 等不支持 system prompt，需要调整指令写法 |
| **Embeddings** | OpenAI Embeddings API 可通过 MCP 服务器或自定义工具接入 |

> **注意**：OpenAI 模型版本与定价以 [OpenAI 官方文档](https://platform.openai.com/docs/models) 为准。本书示例中的模型标识符仅反映撰写时的版本。

### 16.3.2 本地模型：Ollama 接入

Ollama 是主流的本地模型运行平台，支持 Llama、Mistral、Qwen、DeepSeek 等开源模型。本地部署的优势是数据不出本地网络、无 API 调用费用，适合对隐私敏感或网络受限的场景。

#### 接入原理

Ollama 提供兼容 OpenAI API 格式的本地服务（默认端口 `11434`）。OpenClaw 通过 `openai` 供应商类型 + 自定义 `baseUrl` 接入：

```javascript
{
  models: {
    providers: {
      ollama: {
        type: "openai-compatible",
        baseUrl: "http://localhost:11434/v1",
        apiKey: "ollama",  // Ollama 不验证 key，填任意值即可
      },
    },
  },
  agents: {
    defaults: {
      model: {
        primary: "ollama/qwen3:14b",
      },
    },
  },
}
```

#### 启动 Ollama 并拉取模型

```bash
# 安装 Ollama（macOS）
brew install ollama

# 启动服务
ollama serve

# 拉取模型（以 Qwen3 14B 为例）
ollama pull qwen3:14b

# 验证服务
curl http://localhost:11434/api/tags
```

#### 本地模型选型参考

| 模型 | 参数量 | 适用场景 | 显存需求 |
|------|--------|----------|----------|
| `llama3.2:3b` | 3B | 快速问答、简单分类 | ~4GB |
| `qwen3:14b` | 14B | 通用对话、中文支持 | ~10GB |
| `deepseek-r1:32b` | 32B | 复杂推理、代码生成 | ~20GB |
| `mistral:7b` | 7B | 英文对话、工具调用 | ~6GB |

> 实际显存占用取决于量化精度（Q4/Q8 等），以 `ollama show <model>` 输出为准。

### 16.3.3 多生态混合策略

在实际生产中，不同任务对模型能力和成本的要求不同，混合使用多生态模型可以兼顾质量与效率。

#### 典型混合配置

```javascript
{
  agents: {
    // 通用对话：用本地模型降低成本
    chat: {
      model: {
        primary: "ollama/qwen3:14b",
        fallbacks: ["openai/gpt-4o-mini"],
      },
    },
    // 复杂推理与代码生成：用 Claude Sonnet
    dev_assistant: {
      model: {
        primary: "anthropic/claude-sonnet-4-6",
        fallbacks: ["openai/gpt-4o"],
      },
    },
    // 高并发轻量任务：用 OpenAI 小模型
    classifier: {
      model: {
        primary: "openai/gpt-4o-mini",
      },
    },
  },
}
```

#### 混合策略选型参考

| 策略 | 主模型 | Fallback | 适用场景 |
|------|--------|----------|----------|
| 隐私优先 | 本地（Ollama） | `anthropic/claude-haiku-4-5` | 敏感数据不出本地 |
| 成本优先 | `openai/gpt-4o-mini` | 本地（Ollama） | 高并发、低复杂度 |
| 质量优先 | `anthropic/claude-sonnet-4-6` | `openai/gpt-4o` | 推理质量要求高 |
| 离线兜底 | `anthropic/claude-sonnet-4-6` | 本地（Ollama） | 网络不稳定环境 |

Fallback 链路的触发条件和行为详见 [4.4 模型降级与 Fallback 策略](../04_config_models/4.4_failover.md)。

### 16.3.4 统一管理多生态模型

无论接入哪家供应商，OpenClaw 的模型管理界面提供统一视图。可通过以下命令查看当前所有已配置模型的状态：

```bash
openclaw models status --check
```

建议在混合部署中：
1. 为每个供应商配置独立的 `apiKey` 环境变量，避免密钥混用。
2. 对本地模型定期运行探针，确认 Ollama 服务存活。
3. 将各供应商的配额告警阈值统一纳入监控，防止某一路径静默失效。
