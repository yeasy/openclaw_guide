## 14.3 OpenClaw 的用量观测与预算控制

本节介绍如何通过 OpenClaw 内置命令、Dashboard 的 `Control -> Usage` 页面与相关配置实现**实时用量监控**与**精细化预算控制**，防止成本失控。

## 14.3.1 核心观测命令

OpenClaw 提供多个内置命令用于实时观测与诊断。若你更习惯图形化视图，当前 Dashboard 也提供 `Control -> Usage` 页面，可按时间范围查看 Token、成本与会话分布；如果你已经在对话里通过 `/usage cost` 发现异常，再回到 Usage 页做系统级排查通常更高效。

### `/status` 命令

查看当前会话或系统的实时状态。

```bash
/status
```

输出示例：
```yaml
OpenClaw Gateway Status (build: 2026.03.15)

会话统计：
  活跃会话: 42
  Token 使用 (今日): 245.6K / 500K (49%)

成本（今日）:
  LLM: $7.35
  工具: $2.10
  存储: $0.05
  总计: $9.50 / $50.00 预算 (19%)
```

#### `/usage cost` 命令

聊天会话内的本地成本摘要，适合快速看最近一段时间的费用分布。若需要 CLI 视角的完整使用量信息，可配合 `openclaw status --usage`；若需要图形化视图，可对照 Dashboard 的 `Control -> Usage` 页面。

```bash
/usage cost
```

输出：
```text
成本分析（最近 24 小时）

按 Agent 统计：
  support_agent: $23.45 (65%)
  data_analyst_agent: $10.20 (28%)
  routing_agent: $2.15 (6%)
  其他: $0.20 (1%)

按模型统计：
  Claude Sonnet 4.6: $28.50 (79%)
  Claude Haiku 4.5: $7.50 (21%)

按操作统计：
  LLM 推理: $28.00 (78%)
  API 调用: $6.50 (18%)
  存储: $1.50 (4%)
```

#### `/compact` 命令

手动触发上下文压缩，回收 Token 预算。

```bash
/compact --session=<session_id> --target-tokens=5000
```

输出：
```text
压缩完成：
  原始 Token 数: 12350
  压缩后 Token 数: 4820 (61% 回收)
```

### 14.3.2 预算配置与告警

#### 全局预算限制

> [!CAUTION]
> 下列配置示例反映治理架构。具体字段名与 schema 可能因版本而异，请参考当前部署的 OpenClaw 文档。

```json
{
  "budget": {
    "daily": {
      "tokens": 1000000,
      "cost": 500,
      "requests": 100000
    },
    "monthly": {
      "tokens": 30000000,
      "cost": 15000
    }
  },
  "alerts": {
    "enabled": true,
    "thresholds": {
      "dailyTokens": 0.8,
      "dailyCost": 0.9,
      "monthlyTokens": 0.75
    },
    "channels": ["email", "slack"]
  }
}
```

**配置解读**：
- 日预算：100 万 Token 或 $500（先达到的限制）
- 告警触发条件：日 Token 达 80%、日成本达 90%、月 Token 达 75%

#### 按 Agent 预算分配

同理，下面更适合表达“按 Agent 做预算配额”的目标，而不是当前实例已验证通过的字段名：

```json
{
  "agents": {
    "support_agent": {
      "budget": {
        "dailyTokens": 500000,
        "dailyCost": 250,
        "enforceLimit": true
      }
    },
    "research_agent": {
      "budget": {
        "dailyTokens": 200000,
        "dailyCost": 100,
        "enforceLimit": false
      }
    }
  }
}
```

### 14.3.3 上下文管理机制

#### 自动压缩与清理

```json
{
  "context": {
    "compaction": {
      "enabled": true,
      "triggerThreshold": 8000,
      "targetTokens": 5000,
      "preserveRecentMessages": 5,
      "summarizationModel": "claude-haiku-4-5",
      "schedule": "on-demand"
    },
    "expiration": {
      "maxAge": "7d",
      "idleAge": "24h",
      "action": "archive"
    }
  }
}
```

参数语义见 JSON 注释。

#### 会话重置与清理

```bash
# 主动重置某个会话（清空上下文，开始新对话）
/reset --session=<session_id>

# 批量清理过期会话
openclaw cleanup --older-than=30d --dry-run
# （使用 --dry-run 预览要删除的内容，不实际删除）

# 实际执行清理
openclaw cleanup --older-than=30d
```

### 14.3.4 成本控制的工程策略

#### 策略 1：分层模型选择

根据任务复杂度自动选择模型，降低平均成本：

```json
{
  "routing": {
    "complexity_detection": {
      "enabled": true,
      "rules": [
        {
          "condition": { "complexity": "simple" },
          "model": "claude-haiku-4-5",
          "criteria": "查询字数 < 50 且不涉及推理"
        },
        {
          "condition": { "complexity": "medium" },
          "model": "claude-sonnet-4-6",
          "criteria": "一般问题"
        },
        {
          "condition": { "complexity": "high" },
          "model": "claude-opus-4-6",
          "criteria": "多步推理、代码编写、深度分析"
        }
      ]
    }
  }
}
```

实测可降低 55% 成本。

#### 策略 2：工具缓存与重复消除

对重复的工具调用进行缓存，命中率 30-50%（`excludeTools` 中列出不可缓存的工具，如发邮件）：

```json
{
  "tools": {
    "caching": {
      "enabled": true,
      "ttl": 3600,
      "keyStrategy": "hash(tool, args)",
      "excludeTools": ["send_email", "create_ticket"]
    }
  }
}
```

### 14.3.5 成本异常检测与预警

通过日志、会话用量与监控实现异常消费告警。配置示例：

```json
{
  "anomalyDetection": {
    "enabled": true,
    "baselines": {
      "averageDailyCost": 500,
      "stdDeviation": 100
    },
    "triggers": [
      {
        "name": "spike_detection",
        "condition": "dailyCost > baseline + 3 * stdDev",
        "action": "alert + pause_non_critical"
      },
      {
        "name": "model_drift",
        "condition": "error_rate > 5%",
        "action": "notify_admin"
      }
    ]
  }
}
```

### 14.3.6 用量观测与预算控制检查清单

部署 OpenClaw 时，确保完成以下配置：

- **日预算设置**：配置全局日成本预算并按 Agent 分配
- **告警配置**：启用通知渠道（Slack/邮件），设置合理阈值（如 80%）
- **自动压缩**：启用 `context.compaction`，推荐 triggerThreshold 8K、targetTokens 5K
- **会话清理**：配置 `expiration` 策略并定期执行 `openclaw cleanup`
- **成本分析**：每周查看 `/usage cost` 或运行 `openclaw status --usage`，识别最耗资源的 Agent 与操作
- **模型分层**：为不同复杂度的任务选择合适的模型或启用自动复杂度检测

完成以上配置，可实现**可观、可控、可优化**的成本管理体系。
