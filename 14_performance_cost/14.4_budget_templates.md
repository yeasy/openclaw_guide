## 14.4 不同规模部署预算模板

> **请注意**：以下预算模板主要使用 Claude Haiku 4.5、Sonnet 4.6、Opus 4.6 的基础价做示意。若混用 GPT-5.4 等 OpenAI 模型，请以各自官方 pricing 页面为准，不要直接套用本文表格中的固定单价。

本节为三种典型的 OpenClaw 部署规模提供**成本要素分解与预算模板**，帮助你根据实际场景估算成本与规划基础设施。

### 14.4.1 个人/小团队场景

**用户群体**：个人开发者、创业公司初期（<10 人）、副业项目

**假设**：
- 日均用户数：10-50，日均请求数：50-200
- 核心用途：客服助手、内容生成、数据处理
- 部署方式：云 SaaS

#### 个人/小团队场景预算模板

> [!NOTE]
> 下列“模板”用于估算预算规模与治理目标，**不等于当前版本可直接粘贴进 `openclaw.json` 的真实字段**。像 `budget`、`model_selection`、`model_routing`、`context_max_tokens`、`daily_token_limit`、`rate_limiting` 这类写法，在不同版本里可能只是思路占位而非现行 schema。

```json
{
  "budget": {
    "monthly": 50,
    "daily": 1.5,
    "alerts": {
      "threshold_percent": 75,
      "action": "warn_only"
    }
  },
  "model_selection": {
    "default_model": "claude-haiku-4-5",
    "fallback_model": "claude-sonnet-4-6"
  },
  "cost_controls": {
    "context_max_tokens": 5000,
    "output_max_tokens": 1000,
    "daily_token_limit": 300000
  }
}
```

#### 个人/小团队场景优化建议

1. **使用 Haiku 为主**（70-80%）处理简单任务，成本最优
2. **启用消息压缩**，减少存储成本
3. **工具缓存**，降低 API 调用成本

### 14.4.2 中等规模场景

**用户群体**：初创公司（10-50 人）、部门级应用（500-2000 用户）

**假设**：
- 月活用户：500-2000，日均请求数：2000-10000
- 核心用途：企业客服、内容团队支撑、数据分析辅助
- 部署方式：自托管（AWS EC2 或 Google Cloud）+ 部分 SaaS

```json
{
```

#### 中等规模场景优化建议

1. **模型分层**：根据任务复杂度选择不同模型，平衡成本与质量
2. **工具缓存**：Redis 缓存热点查询结果，减少 API 调用 40%
3. **数据库优化**：使用查询索引、批处理减少数据库调用

### 14.4.3 企业规模场景

**用户群体**：大型企业、SaaS 平台、对外服务

**假设**：
- 月活用户：10K-100K+，日均请求数：100K-1M
- 核心用途：客户服务、销售支撑、内部自动化
- 部署方式：专用基础设施 + 混合云

#### 企业规模场景优化建议

1. **智能路由与地域感知**：根据用户位置选择最近的模型 API，减少延迟和成本
2. **分级缓存**：L1 内存、L2 Redis、L3 CDN，最小化数据库与 API 调用
3. **请求去重**：识别并合并重复请求，避免重复计费
4. **成本优化合同**：与模型供应商谈判体积折扣（>10B tokens/月 通常可获 10-20% 优惠）

### 14.4.4 成本驱动因素对比

| 驱动因素 | 个人 | 中等 | 企业 |
|---------|------|------|------|
| LLM 成本占比 | 2% | 1% | 0% |
| 工具成本占比 | 70% | 48% | 16% |

随规模增长，成本结构从工具/托管主导逐步转向基础设施、再转向人力成本主导。

### 14.4.5 预算规划决策树

```text
你想部署 OpenClaw 吗？
│
├─ 预期日均请求 < 100？
│  └─ 选择个人模板（$30-60/月）
│     ├─ 部署方式：云 SaaS

│
├─ 预期日均请求 100-10K？
│  └─ 选择中等模板（$500-800/月）
│     ├─ 部署方式：自托管或 VPS
│     └─ 模型选择：分层路由
│
└─ 预期日均请求 > 10K？
   └─ 选择企业模板（$10K-50K/月）
      ├─ 部署方式：专用多区域基础设施
      ├─ 模型选择：高级路由 + 微调
      └─ 额外需求：监控、告警、灾难恢复
```
