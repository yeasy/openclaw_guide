## 14.2 不同规模部署的成本预算与优化路径

本节为个人用户、团队和企业三个不同规模的OpenClaw部署场景提供具体的成本预算模型和优化策略。

### 14.2.1 成本要素分解

OpenClaw 的运营成本主要包括：

```text
总成本 = API 成本 + 基础设施成本 + 人工成本 + 维护成本

1. API 成本（通常占 60-70%）
   ├─ LLM 推理成本（Claude/GPT调用）
   ├─ 第三方工具 API 成本（GitHub, Slack, Zendesk等）
   └─ 向量化存储与检索（知识库）

2. 基础设施成本（通常占 20-25%）
   ├─ 计算资源（CPU/内存）
   ├─ 存储（日志、缓存、数据库）
   └─ 网络带宽

3. 人工成本（通常占 5-10%）
   ├─ 初期部署与配置
   ├─ 日常维护与监控
   └─ 知识库更新与优化

4. 维护成本（通常占 2-5%）
   ├─ 安全更新与补丁
   ├─ 故障排查与优化
   └─ 合规与审计
```

**1. 混合模型智能路由**

```javascript
// 根据请求复杂度动态选择模型
class SmartModelRouter {
  selectModel(request) {
    const complexity = this.estimateComplexity(request);
    const cached = this.checkCache(request);

    if (cached) {
      return { model: 'cached', cost: 0, latency: 5 };
    }

    if (complexity < 0.3) {
      // 简单请求：使用Haiku ($0.001/req)
      return { model: 'haiku', cost: 0.001, latency: 500 };
    } else if (complexity < 0.7) {
      // 中等请求：使用Sonnet ($0.02/req)
      return { model: 'sonnet', cost: 0.02, latency: 1000 };
    } else {
      // 复杂请求：使用Opus ($0.08/req)
      return { model: 'opus', cost: 0.08, latency: 1500 };
    }
  }

  estimateComplexity(request) {
    const factors = [
      { weight: 0.3, value: request.toolCount / 10 }, // 工具使用数
      { weight: 0.3, value: request.contextSize / 5000 }, // 上下文大小
      { weight: 0.2, value: request.turnCount / 20 }, // 对话轮数
      { weight: 0.2, value: request.hasComputations ? 1 : 0 } // 是否需要计算
    ];

    return factors.reduce((sum, f) => sum + f.weight * f.value, 0);
  }
}

// 预期成本节省：40-60%（通过避免使用昂贵模型）
```

**3. 请求批处理**

```javascript
// 批量处理类似请求，减少API调用次数
class BatchProcessor {
  private queue = [];
  private batchSize = 10;
  private batchTimeout = 5000; // 5秒

  async addRequest(request) {
    this.queue.push(request);

    if (this.queue.length >= this.batchSize) {
      return this.processBatch();
    }

    return new Promise(resolve => {
      const timer = setTimeout(() => {
        if (this.queue.length > 0) {
          this.processBatch().then(resolve);
        }
      }, this.batchTimeout);
    });
  }

  private async processBatch() {
    const requests = this.queue.splice(0, this.batchSize);

    // 合并相似请求，减少LLM调用
    const uniqueRequests = this.deduplicateSimilar(requests);

    const results = await llm.batchComplete(uniqueRequests);

    // 映射结果回原始请求
    return this.mapResultsBack(requests, results);
  }

  private deduplicateSimilar(requests) {
    const unique = [];
    const seen = new Set();

    for (const req of requests) {
      const hash = this.getRequestHash(req);
      if (!seen.has(hash)) {
        unique.push(req);
        seen.add(hash);
      }
    }

    return unique;
  }
}

// 成本节省：15-25%（通过批处理）
```

**2. 自建向量数据库与检索优化**

```javascript
// 自建向量DB而非依赖第三方服务
class EnterpriseVectorStore {
  constructor() {
    this.qdrant = new QdrantClient({
      url: 'http://qdrant-cluster:6333',
      api_key: process.env.QDRANT_API_KEY
    });
    this.cache = new RedisCluster({
      nodes: ['redis1:6379', 'redis2:6379', 'redis3:6379']
    });
  }

  async search(query, topK = 5) {
    // 1. 先查Redis缓存（L1缓存，<1ms）
    const cached = await this.cache.get(`search:${hash(query)}`);
    if (cached) return cached;

    // 2. 查向量DB（L2缓存，<100ms）
    const vector = await this.encoder.encode(query);
    const results = await this.qdrant.search(vector, topK);

    // 3. 缓存结果
    await this.cache.setex(
      `search:${hash(query)}`,
      3600, // 1小时过期
      JSON.stringify(results)
    );

    return results;
  }

  // 成本对比
  // 第三方服务：$5000/月 + API调用成本
  // 自建方案：$2000/月（基础设施）+ 人工成本
  // 预期节省：$2000-3000/月
}
```

```javascript
class MonthlyFinanceReview {
```
