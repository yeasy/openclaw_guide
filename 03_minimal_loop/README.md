# 第三章 快速上手与首轮对话实战

本章建立可复验的”最小可用基准线”：先用控制台与 WebChat 验证可用的本地主链路，掌握诊断命令与排障顺序；再固化初始指令目标与格式；最后为入口建立最小安全与访问边界。通过本章，你将获得第一个可工作的 OpenClaw 实例，并为后续深入学习建立扎实基础。

## 本章内容导读

本章包括以下几个小节：

- **[3.1 控制台与 WebChat 快速上手](3.1_control_ui_webchat.md)**：用 Dashboard 与 WebChat 验证可用的本地对话基线，学会用日志解释每一步发生了什么。
- **[3.2 常用诊断命令与日志排障](3.2_diagnostics.md)**：形成固定的诊断顺序与证据链，避免”凭感觉改配置/改提示词”。
- **[3.3 初始指令与智能体角色配置](3.3_agent_persona.md)**：写出”可执行的初始指令”：目标收敛、边界声明、输出结构约束。
- **[3.4 配对、群聊策略与访问边界](3.4_pairing_groups.md)**：为入口建立最小信任边界：配对审批、群聊门控、允许列表与最小权限联动。
- **[3.5 本章小结](summary.md)**：关键要点与自测题。

## 学习目标

完成本章的阅读后，你将能够：
1. **快速验证**：用 Dashboard 与 WebChat 验证可用的对话基线。
2. **自我诊断**：掌握常用的诊断命令与排障思路。
3. **编写指令**：设计清晰、可执行的初始指令与角色定义。
4. **建立边界**：实施最小权限原则，为系统建立访问控制。

## 前置条件

- 已完成[第二章](../02_setup/README.md)的安装与首跑验证。
- 可打开本机浏览器访问 Dashboard/WebChat（远程环境需端口转发）。
