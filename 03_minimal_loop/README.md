# 第三章：快速上手与首轮对话实战

本章建立可复验的“最小可用基准线”：先用控制台与 WebChat 跑通本地主链路，再固化初始指令与访问边界，最后把诊断命令与排障顺序沉淀为团队可复用的方法。

**本章学习目标**

读完本章后，你应能：

- 用 Dashboard 与 WebChat 跑通可重复的本地对话基线，并能用日志解释每一步发生了什么。
- 写出“可执行的初始指令”：目标收敛、边界声明、输出结构约束。
- 为入口建立最小信任边界：配对审批、群聊门控、允许列表与最小权限联动。
- 形成固定的诊断顺序与证据链，避免“凭感觉改配置/改提示词”。

**前置条件**

- 已完成第二章的安装与首跑验证。
- 可打开本机浏览器访问 Dashboard/WebChat（远程环境需端口转发）。

**实践建议**

建议先把本地基准线做“稳定”，再做“扩展”。多渠道、多账号与自动化会引入大量外部变量；没有基准线时，问题往往难以定界与复现。

推荐阅读顺序（更利于排障与复验）：

- `3.1 → 3.2 → 3.3 → 3.4`
- 可选：完成 3.1–3.4 的基准线后，再做 3.5 飞书接入。
