# 第三章：快速上手与首轮对话实战

本章建立可复验的“最小可用基准线”：先用控制台与 WebChat 跑通本地主链路，掌握诊断命令与排障顺序；再固化初始指令目标与格式；最后为入口建立最小安全与访问边界。

**本章学习目标**

读完本章后，你应能：

- 用 Dashboard 与 WebChat 跑通可重复的本地对话基线，并能用日志解释每一步发生了什么。
- 形成固定的诊断顺序与证据链，避免“凭感觉改配置/改提示词”。
- 写出“可执行的初始指令”：目标收敛、边界声明、输出结构约束。
- 为入口建立最小信任边界：配对审批、群聊门控、允许列表与最小权限联动。

**前置条件**

- 已完成第二章的安装与首跑验证。
- 可打开本机浏览器访问 Dashboard/WebChat（远程环境需端口转发）。
